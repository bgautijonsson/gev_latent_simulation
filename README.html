GEV Latent Simulation

A Bayesian framework for integrating climate model simulations with limited observational data using Generalized Extreme Value (GEV) distributions and latent variable modeling.

Overview

This project implements statistical methods for analyzing extreme climate events by combining sparse observational data (y) with abundant climate model simulations (X). The approach is inspired by the climate modeling frameworks of Rougier and Chandler, addressing the challenge of making inference about extreme events when observational records are limited but simulation data is plentiful.

Key Features

  • Hierarchical Bayesian modeling using Stan for parameter estimation
  • GEV distribution modeling for extreme value analysis
  • Latent variable framework linking simulation parameters to true climate parameters
  • Linear relationship enforcement between simulated and observed parameters

Methodology

The statistical framework assumes:

  1. Few observations y following a GEV distribution with true parameters (μ_true, σ_true, ξ_true)

  2. Many simulations X (matrix of size N_sim × P_sim) from climate models

  3. Linear relationship between simulation and true parameters:

    μ_sim = α_sim + β_sim × μ_true + ε

This hierarchical structure allows the model to: - Borrow strength from simulation data to inform observational inference - Quantify uncertainty in the relationship between simulated and observed extremes - Provide robust estimates of extreme event probabilities

Usage

Basic Example

# Source the main script
source("run_sim.R")

# The script will:
# 1. Generate synthetic observational data (N_obs = 40)
# 2. Create climate model simulations (N_sim × P_sim matrix)
# 3. Fit the hierarchical Bayesian model
# 4. Display parameter estimates and diagnostics

Model Parameters

  • N_obs: Number of observational data points (default: 40)
  • N_sim: Number of simulations per model (default: 100)
  • P_sim: Number of climate model runs (default: 20)
  • mu_true, sigma_true, xi_true: True GEV parameters for observations
  • alpha_sim, beta_sim, sigma_sim: Relationship parameters linking simulations to truth

Key Model Components

  1. Observational Model:

    y_i ~ GEV(μ_true, σ_true, ξ_true)

  2. Simulation Model:

    X_ij ~ GEV(μ_sim[j], σ_true, ξ_true)
μ_sim[j] = α_sim + β_sim × μ_true + σ_sim × z_j

  3. Priors:

    • α_sim ~ Normal(0, 1)
    • β_sim ~ Normal(0, 1)
    • σ_sim ~ Normal(0, 1)

Statistical Background

Generalized Extreme Value Distribution

The GEV distribution is the limiting distribution of block maxima and is parameterized by: - Location (μ): Central tendency of extremes - Scale (σ): Variability of extremes
- Shape (ξ): Tail behavior (ξ < 0 for bounded tail, ξ > 0 for heavy tail)

Latent Variable Framework

The model treats the true climate parameters as latent variables that must be inferred from both: - Limited observational data providing direct but sparse information - Abundant simulation data providing indirect but plentiful information

This approach is particularly valuable for: - Climate change studies where historical records are short - Extreme event analysis where rare events are poorly sampled - Multi-model ensemble analysis incorporating multiple climate projections

Model Output

The fitted model provides estimates and uncertainty quantification for:

  • μ_true, σ_true, ξ_true: True GEV parameters for observations
  • α_sim, β_sim, σ_sim: Parameters linking simulations to observations
  • Model diagnostics and convergence statistics

Example output:

fit$summary(c("mu_true", "sigma_true", "xi_true", "alpha_sim", "beta_sim", "sigma_sim"))

Applications

This framework is suitable for:

  • Climate impact assessment using limited observational records
  • Extreme event attribution combining models and observations
  • Future projection of extreme events under climate change
  • Multi-model ensemble analysis and uncertainty quantification
  • Infrastructure design incorporating climate model information
